## ✅ Fixture 07：CSV 编码与分隔符样例（5 个变体 × 1 万行）

**文件**：`testdata_smoke/07_CSV_*.csv` 共 5 个

| 文件 | 编码 | 分隔符 | 场景 |
|---|---|---|---|
| `07_CSV_UTF8_逗号_1万行.csv` | UTF-8 无 BOM | `,` | Linux / Web 导出最常见 |
| `07_CSV_UTF8BOM_逗号_1万行.csv` | UTF-8 带 BOM | `,` | Windows Excel 另存 CSV 的默认 |
| `07_CSV_GBK_逗号_1万行.csv` | GBK | `,` | 老版 WPS / 国内老业务系统导出 |
| `07_CSV_UTF8_分号_1万行.csv` | UTF-8 | `;` | 欧洲地区（法、德、比利时） |
| `07_CSV_UTF8_Tab_1万行.csv` | UTF-8 | `\t` | TSV 风格 / 数据库导出 |

**所有文件共享同一套 6 列**：订单号 / 客户类型 / 产品 / 数量 / 单价 / 备注

**埋的关键词**：
- `VIP 客户`（每个文件约 **2700 行**；`customers[1]` 每 4 行一个 + `i%50==0` 强制埋点）
- `退货`（每个文件约 **1446 行**；`notes[3]` 每 7 行一个 + `i%500==0` 强制埋点）

### ⚠️ 重要前提：这些文件**不是给 Excel 双击打开看的**

直接用 Excel / WPS 双击 GBK 文件会乱码、双击分号文件会"所有内容挤在 A 列"——**这是 Excel 的行为，不是文件的 bug**。Excel 双击打开 CSV 时只按 UTF-8 解码 + 逗号分隔，不嗅探 GBK 编码和其他分隔符。

**fixture 07 的测试对象是 `excel-master.exe` 能否正确读取**，不是 Excel 能否直接显示。验证 Excel 显示没意义。

### 测试步骤

| # | 操作 | 期望 |
|---|---|---|
| 1 | **批量提取 → 单文件** → 选 `07_CSV_UTF8_逗号_1万行.csv` | 预览：6 列表头 |
| 2 | 关键词 `VIP 客户` → 输出合成 1 个 xlsx | 命中 ~2700 行 |
| 3 | 换 `07_CSV_UTF8BOM_逗号_1万行.csv` 重复 | 命中数与上一步相同（BOM 被正确剥离） |
| 4 | 换 `07_CSV_GBK_逗号_1万行.csv` 重复 | 命中数相同，输出 xlsx 里中文**显示正确**（关键：GBK 被正确识别并转码 UTF-8）|
| 5 | 换 `07_CSV_UTF8_分号_1万行.csv` → **设置 → CSV 分隔符**选"分号" | 命中 ~2700 行（若不选分隔符会报错或只命中 0 行）|
| 6 | 换 `07_CSV_UTF8_Tab_1万行.csv` → **设置 → CSV 分隔符**选"Tab" | 命中 ~2700 行 |
| 7 | 日志里应显示 `[TIMING] 打开 CSV ... (utf-8 / utf-8-bom / gb18030)` | 编码嗅探结果正确 |
| 8 | **批量提取 → 文件夹**选整个 `testdata_smoke/` | 混源场景：5 个 CSV + 01-06 的 xlsx 能同时处理，不报错；只搜 `VIP 客户` 能命中 CSV 和 xlsx 的行并合并输出 |

### 性能基线

- 单个 CSV（1 万行）提取 **≤ 1 秒**（纯文本流式读速度非常快）
- 5 个 CSV 混合批量提取 **≤ 3 秒**

### 常见"看似 bug"但实际不是的情况

| 现象 | 真相 |
|---|---|
| Excel 双击 GBK 文件乱码 | Excel 默认按 UTF-8 解码 → 正常，不是软件 bug |
| Excel 双击分号 / Tab 文件所有内容挤 A 列 | Excel 双击 CSV 只认逗号 → 正常，需要"数据 → 从文本"手动选分隔符才能正确显示 |
| `excel-master.exe` 不选分隔符直接跑分号文件 → 0 命中 | 项目的自动嗅探只支持逗号（ExtractTask.CSVDelimiter 字段就是给这用的），**必须在界面勾选对应分隔符** |

## 🚨 通用红线（任何 fixture 测试都该守）

不论用哪个 fixture，下面任意一项发生 = bug：

| 红线 | 含义 |
|---|---|
| Excel 打开报"部分内容有问题" | 输出文件结构损坏 |
| 命中数据少于埋点 50%+ | 数据丢失 bug |
| 内存涨到 1GB+ | 流式失效，OOM 风险 |
| 单文件提取超过 2 分钟 | 性能严重退化（应在 10 秒内） |
| 进程崩溃 / 黑屏 | 严重 bug |
| 任务取消按钮没反应 | 取消机制坏了 |
