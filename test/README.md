# VBA vs Go 程序 性能对比

这里放两个 VBA 宏，用来跟「Excel 拆合大师」的"单文件提取"模式做性能对比。

## 两个脚本的定位

| 脚本 | 用途 | 复制内容 | 速度 | 对比公平度 |
|---|---|---|---|---|
| `ExtractByKeyword.bas` | 理论最快极限值 | 只复制**值** | 最快 | ❌ 不公平（VBA 干的事少）|
| `ExtractByKeywordFull.bas` | **真实使用场景** | 值 + 公式 + 格式 + 行高列宽 + **图片** | 慢 | ✅ 跟 Go 程序"保留图片跟随行"对齐 |

**推荐测试顺序**：先跑 `ExtractByKeywordFull.bas` 看真实对比；再跑 `ExtractByKeyword.bas` 看 VBA 理论极限。

## 行为对齐表（ExtractByKeywordFull.bas）

| 维度 | Excel 拆合大师（Go，单文件提取 + 保留图片）| VBA 完整版 |
|---|---|---|
| 处理对象 | 单文件模式选中的 xlsx | ActiveWorkbook |
| 扫描范围 | 全部 Sheet（除非手动选）| 全部 Sheet |
| 表头 | 第 1 行 | 第 1 行 |
| 搜索 | 全列搜索 | 全列搜索（InStr 包含）|
| 匹配 | 精准 + 包含 | 包含（VBA InStr）|
| 输出 | 新文件（zip 手术，100% 保真）| 新文件（Excel SaveAs）|
| 保留列宽 | ✅ | ✅ |
| 保留行高 | ✅ | ✅ |
| 保留公式 | ✅ | ✅（PasteSpecial xlPasteAll）|
| 保留样式 | ✅（字节级克隆）| ✅（剪贴板复制）|
| 保留合并单元格 | ✅ | ✅ |
| **保留图片跟随行** | ✅ | ✅（Shape.Copy）|
| 输出方式 | 新文件 / 写回新 Sheet | 同左（弹窗选）|

## 用法

### 步骤（以完整版为例）

1. 打开你要测试的源 xlsx 文件（双击它，让它成为 ActiveWorkbook）
2. `Alt+F11` 打开 VBA 编辑器
3. **文件 → 导入文件** → 选 `ExtractByKeywordFull.bas`
4. 按 `F5` 或工具栏运行，执行 `ExtractByKeywordFull` 子过程
5. **弹窗 1**：输入关键词（如 `汉族`）
6. **弹窗 2**：选输出方式
   - **是** = 新文件（自动保存到 `F:\桌面\测试拆分图片\原文件名_VBA完整结果_时间戳.xlsx`）
   - **否** = 写回源文件新 Sheet（完成后请 Ctrl+S）
7. 完成弹窗显示：扫描行数 / 命中行数 / **复制图片数** / 总耗时

### 直接复制粘贴法（不想导入）

1. 打开源 xlsx → `Alt+F11`
2. **插入 → 模块** → 把 `.bas` 全文粘进来（**包括第 1 行的 Attribute**）
3. `F5`

## 公平对比建议

1. **跑同一个文件**：关键词、源文件、期望输出方式保持完全一致
2. **先冷启动再跑第一次**：Excel 和我们的 Go 程序都有冷启动开销
   - 跑 VBA 前：Excel 完全退出再重新打开源文件
   - 跑 Go 程序前：确认 Excel 已关掉源文件（避免文件占用）
3. **至少跑 3 次取后两次的平均**
4. **关注指标**：
   - **总耗时**
   - **命中行数**（应该两边一致）
   - **输出文件大小**（VBA 的 SaveAs 会重写整个 xlsx；Go 程序的 zip 手术通常更小）
   - **图片数**（我们的 Go 程序跟 VBA 完整版都会保留图片）

## 预期结果（实测）

我们的 Go 程序对 2 万行 × 9 列 + 带照片的 xlsx（单文件提取）实测：
- **Go 程序**：约 10 秒（包括打开、扫描、zip 手术、图片迁移）
- **VBA 纯值版**：约 3~5 秒（但没图片、没公式、没格式，不公平）
- **VBA 完整版**：约 30~120 秒（带格式 + 图片逐张 Shape.Copy 极慢）

关键瓶颈：
- VBA 图片复制走 Excel 剪贴板，每张图要 10~50 ms。1 万张图就要 100~500 秒
- VBA PasteSpecial xlPasteAll 也是逐单元格同步样式，比 Go 程序的 zip 手术（字节级克隆）慢几十倍

## 注意事项

- **输出目录硬编码**：完整版默认输出到 `F:\桌面\测试拆分图片`；没有盘符就改 `.bas` 里的 `OUTPUT_DIR` 常量
- **列数以 UsedRange 为准**：如果源文件不同 Sheet 列数不一致，可能跟 Go 程序的"每 Sheet 独立表头"行为略有偏差
- **写回模式跑完不会自动保存**（避免锁文件），需要手动 `Ctrl+S`
- **大文件风险**：源 xlsx >200MB 时，VBA 的 `UsedRange.Value` 可能内存溢出；Go 程序用流式扫描不受此限
- **完整版跑完后清剪贴板**：我们的 `Application.CutCopyMode = False` 会清理，正常情况下不会有大剪贴板残留弹窗
